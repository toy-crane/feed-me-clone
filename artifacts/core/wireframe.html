Notes

앱 첫 진입 화면. 입력 필드만 노출되고 결과 영역은 없음.

  • 트리거: 앱 첫 진입 또는 새로고침 직후
  • 인터랙션: URL 입력 → 화살표 버튼 활성화 → 클릭 시 변환 시작 (또는 Enter)
  • 전환: 변환 성공 시 화면 1a로 (결과 영역이 펼쳐진 형태)
  • 다크모드 토글: 우측 상단. OS 설정 따라가다가 사용자 선택 시 localStorage에 저장

Notes

변환 직후 기본 상태. 결과 헤더(좌측) 옆 우측 상단에 split 버튼. 프롬프트 영역은 접혀 있고 헤더와 본문 사이에 위치.

  • 트리거: 화면 0에서 화살표 클릭 → 변환 성공
  • 인터랙션: split 버튼의 메인 영역 "복사하기" 클릭 → 본문(프롬프트 비었으므로) 복사 + "복사됨" 토스트 / 우측 ∨ 클릭 → 화면 1c처럼 메뉴 노출 / "프롬프트 추가하기" 클릭 → 화면 1b로
  • 전환: 새 URL을 입력하고 화살표 누르면 헤더·본문이 새 데이터로 교체됨. 입력 필드 X 클릭 시 입력만 비고 결과·프롬프트 상태는 유지
  • 모바일에서 split 버튼이 우측 상단 고정 — 헤더 텍스트가 길면 줄바꿈, 버튼은 잘리지 않음

"+ 프롬프트 추가하기" 클릭으로 펼친 상태. textarea 위, 칩 아래 배치. 칩 클릭 시 textarea 내용을 칩 텍스트로 덮어쓰기.

  • 트리거: 화면 1a에서 "+ 프롬프트 추가하기" 클릭
  • 인터랙션: textarea 직접 입력 / 프리셋 칩 클릭 → textarea 덮어쓰기 / 토글 다시 클릭 → 다시 접힘 (textarea 내용은 메모리에 유지되지만 화면에서는 안 보임)
  • 이 상태에서 "복사하기" → 클립보드에 [textarea 내용] + 빈 줄 + 본문
  • 새로고침 시: 펼침 상태와 textarea 내용 모두 초기화 (1회용)

Notes

Split 버튼의 우측 화살표(∨) 클릭으로 메뉴가 열린 상태. 메뉴는 split 버튼 우측 정렬로 떠 있음.

  • 트리거: 화면 1a 또는 1b에서 split 버튼 우측 ∨ 클릭
  • 인터랙션: 메뉴 항목 클릭 → 해당 동작 수행 후 메뉴 닫힘 (마크다운 다운로드 = 본문만 .md 파일 / ChatGPT·Claude 열기 = 클립보드에 [프롬프트(있으면)+본문] 복사 + 새 탭 + 안내 토스트)
  • 메뉴 바깥 클릭 또는 Esc → 메뉴 닫힘
  • 모바일에서 메뉴가 화면 우측 가장자리를 넘어가지 않도록 우측 정렬
