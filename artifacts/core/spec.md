# core — URL → Markdown 변환기

## 개요

웹 페이지 URL을 입력하면 본문을 Markdown으로 추출해 보여주고, 그것을 LLM(ChatGPT·Claude)으로 빠르게 넘기거나 복사·다운로드할 수 있는 단일 페이지 도구. 사용자는 URL을 붙여넣고 화살표 버튼을 눌러 변환된 본문과 내보내기 옵션을 즉시 확인한다.

## 범위

### 포함
- URL 입력 → Markdown 변환 → 본문·헤더·내보내기 옵션을 같은 화면에 표시
- 잘못된 URL 형식 차단, 그 외 변환 실패는 일반 "변환 실패" 메시지
- 본문 복사, `.md` 파일 다운로드, ChatGPT/Claude로 열기 (프롬프트 선택 가능)
- 프롬프트 프리셋 3종 + 직접 입력 (1회용)
- 다크모드 토글 (시스템 기본값 추종, 사용자 선택은 localStorage 유지)

### 제외
- **사용자 계정·로그인** — 누구나 익명 사용. 인증을 도입하면 본 feature의 모든 시나리오가 흔들리므로 별도 feature로 분리한다.
- **변환 히스토리·즐겨찾기** — 페이지 이탈/새로고침 시 결과는 사라진다. 저장이 필요하다는 신호가 사용자에서 들어오면 그때 별도 feature로 다룬다.
- **배치·다중 URL 변환** — 한 번에 한 URL만. 큐·여러 탭 동시 변환은 다루지 않는다.
- **ChatGPT·Claude 입력창에 본문 자동 prefill** — URL 파라미터를 통한 자동 채움은 하지 않는다. 클립보드 복사 + 새 탭으로 일관된 UX를 제공한다 (본문 길이에 따른 분기 제거 → 시나리오 단순화).
- **변환 진행률·예상 시간 표시** — 스피너만으로 충분.

## 시나리오

### 1. URL 변환 성공 → 결과 표시

- **Given** — 빈 입력 필드와 비활성 화살표 버튼이 보이는 초기 화면
- **When** — 사용자가 유효한 웹 페이지 URL을 입력하고 화살표 아이콘 버튼(또는 Enter 키)을 누른다
- **Then** — 화살표 버튼이 스피너로 바뀌었다가, 변환 완료 시 결과 영역에 페이지 제목·저자·출처 URL 헤더와 렌더링된 Markdown 본문이 나타난다

성공 기준:
- [ ] 유효 URL 입력 + 화살표 버튼 클릭 → 결과 영역에 페이지 제목이 큰 헤딩으로 표시된다
- [ ] 페이지에서 저자가 추출된 경우 → 제목 아래 작은 텍스트로 저자 이름이 표시된다
- [ ] 페이지에서 저자가 추출되지 않은 경우 → 저자 줄 자체가 보이지 않는다
- [ ] 입력한 원본 URL이 헤더 영역에 항상 표시된다 (출처 확인용)
- [ ] 본문은 Markdown 문법(헤딩, 리스트, 링크, 코드 블록 등)이 시각적으로 렌더링된 형태로 보인다
- [ ] 변환이 진행되는 동안 화살표 버튼 자리에 스피너가 보이고, 다시 누를 수 없다 (별도 "변환 중..." 텍스트는 표시하지 않는다)

### 2. 잘못된 URL 형식 → 입력 단계 차단

- **Given** — 입력 필드에 빈 문자열 또는 URL 형식이 아닌 텍스트(예: `abc`, `not a url`)가 들어 있는 상태
- **When** — 사용자가 화살표 버튼을 누르거나 Enter 키를 누른다
- **Then** — 변환이 시작되지 않고, 입력 필드 아래에 "올바른 URL을 입력해 주세요" 안내가 보인다

성공 기준:
- [ ] 빈 문자열 → 변환이 시작되지 않고 안내 문구가 보인다
- [ ] `abc` 같은 비-URL 텍스트 → 변환이 시작되지 않고 안내 문구가 보인다
- [ ] 사용자가 입력 내용을 수정하면 안내 문구가 사라진다
- [ ] 화살표 버튼은 입력이 비어 있는 동안 비활성 상태로 보인다

### 3. 변환 실패 → 일반 실패 메시지

- **Given** — URL 형식은 유효하지만 페이지에 도달할 수 없거나 본문 추출에 실패하는 URL이 입력된 상태
- **When** — 사용자가 화살표 버튼을 누른다
- **Then** — 스피너가 사라지고 "변환에 실패했어요. 잠시 후 다시 시도해 주세요" 메시지가 표시되며, 입력값은 그대로 남아 사용자가 즉시 다시 시도하거나 다른 URL로 바꿀 수 있다

성공 기준:
- [ ] 도달 불가 URL(예: 존재하지 않는 도메인) → "변환에 실패했어요…" 메시지가 표시된다
- [ ] 본문이 추출되지 않는 페이지 → 동일한 "변환에 실패했어요…" 메시지가 표시된다 (실패 원인을 사용자에게 분기해 보여주지 않는다)
- [ ] 실패 후에도 입력 필드의 URL은 유지된다
- [ ] 실패 후 다시 화살표 버튼을 누르면 변환이 재시도된다

### 4. 결과를 클립보드에 복사

- **Given** — 변환 결과가 화면에 표시된 상태
- **When** — 사용자가 결과 영역의 "복사하기" 버튼을 누른다
- **Then** — 변환된 Markdown 본문(프롬프트는 포함하지 않음)이 클립보드에 복사되고, "복사됨" 토스트가 잠시 보인다

성공 기준:
- [ ] "복사하기" 버튼 클릭 → 클립보드 내용이 화면에 보이는 Markdown 본문 텍스트와 일치한다
- [ ] 프롬프트 프리셋 또는 직접 입력 텍스트가 선택되어 있어도 → 클립보드에는 본문만 들어간다 (프롬프트 미포함)
- [ ] 클릭 직후 "복사됨" 안내가 잠시 표시되고 사라진다

### 5. `.md` 파일 다운로드

- **Given** — 변환 결과가 화면에 표시된 상태
- **When** — 사용자가 "다운로드" 버튼을 누른다
- **Then** — 페이지 제목을 파일명으로 한 `.md` 파일이 사용자 기기로 다운로드된다

성공 기준:
- [ ] 페이지 제목이 `Hello, World!`라면 → `Hello__World_.md` 형태(파일명에 못 쓰는 문자는 `_`로 치환된 형태)로 다운로드된다
- [ ] 페이지 제목 추출에 실패한 경우 → `page-<YYYYMMDD-HHMM>.md` 형태(예: `page-20260429-1430.md`)로 다운로드된다
- [ ] 파일 내용은 화면에 보이는 Markdown 본문 텍스트와 일치한다 (프롬프트 미포함)

### 6. ChatGPT·Claude로 열기 (프롬프트 없음)

- **Given** — 변환 결과가 표시되어 있고, 프롬프트 영역에서 어떤 프리셋도 "직접 입력"도 선택되지 않은 상태
- **When** — 사용자가 "ChatGPT로 열기" 또는 "Claude로 열기" 버튼을 누른다
- **Then** — Markdown 본문이 클립보드에 복사되고 해당 LLM 웹 인터페이스가 새 탭으로 열리며, "클립보드에 복사했어요. 입력창에 붙여넣어 주세요" 안내가 화면에 보인다

성공 기준:
- [ ] "ChatGPT로 열기" 클릭 → 새 탭으로 ChatGPT 웹 페이지가 열린다
- [ ] "Claude로 열기" 클릭 → 새 탭으로 Claude 웹 페이지가 열린다
- [ ] 클립보드 내용은 Markdown 본문 텍스트와 일치한다 (앞에 프롬프트 없음)
- [ ] "클립보드에 복사했어요. 붙여넣어 주세요" 안내가 화면에 표시된다

### 7. ChatGPT·Claude로 열기 (프롬프트 적용)

- **Given** — 변환 결과가 표시되어 있고, 프롬프트 영역에서 프리셋 1개(예: `요약해줘`) 또는 "직접 입력"의 텍스트가 채워진 상태
- **When** — 사용자가 "ChatGPT로 열기" 또는 "Claude로 열기" 버튼을 누른다
- **Then** — `<선택된 프롬프트>\n\n<Markdown 본문>` 형태의 텍스트가 클립보드에 복사되고, LLM 웹 인터페이스가 새 탭으로 열리며 안내 토스트가 표시된다

성공 기준:
- [ ] 프리셋 `요약해줘` 선택 + ChatGPT로 열기 → 클립보드 첫 줄이 `요약해줘`로 시작하고, 한 줄 빈 행 뒤에 본문이 이어진다
- [ ] "직접 입력"에 `존댓말로 풀어줘` 입력 + Claude로 열기 → 클립보드 첫 줄이 `존댓말로 풀어줘`로 시작하고, 빈 행 뒤에 본문이 이어진다
- [ ] 프롬프트가 적용된 LLM 열기 시에도 결과 화면의 본문 영역과 "복사하기"·"다운로드" 동작은 영향을 받지 않는다 (프롬프트는 LLM 열기에만 적용)

### 8. URL 입력 지우기

- **Given** — 입력 필드에 URL이 들어 있고 결과 영역에 변환 결과가 표시된 상태
- **When** — 사용자가 입력 필드 안의 X 아이콘을 누른다
- **Then** — URL 입력 필드만 비워지고, 결과 영역과 프롬프트 선택 상태는 그대로 유지된다

성공 기준:
- [ ] X 아이콘 클릭 → 입력 필드가 빈 상태가 된다
- [ ] X 아이콘 클릭 → 결과 영역의 헤더·본문은 그대로 보인다
- [ ] X 아이콘 클릭 → 프롬프트 영역의 선택 상태는 그대로 유지된다
- [ ] 입력 필드가 빈 동안 X 아이콘은 보이지 않는다

### 9. 다크모드 토글

- **Given** — 화면 우측 상단에 다크모드 토글 아이콘이 있는 어떤 상태든
- **When** — 사용자가 토글을 누른다
- **Then** — 라이트/다크 테마가 즉시 전환되고, 다음 방문 시에도 사용자가 선택한 테마가 적용된다

성공 기준:
- [ ] 첫 방문 시 → OS 시스템 설정(`prefers-color-scheme`)에 맞는 테마로 보인다
- [ ] 토글 클릭 → 페이지 전체 색상이 즉시 반대 테마로 바뀐다
- [ ] 페이지를 새로고침하거나 다음 날 다시 방문 → 마지막에 선택한 테마가 그대로 적용된다 (시스템 설정 변경과 무관)

## 불변 규칙

### 프롬프트 적용 범위
프롬프트(프리셋·직접 입력)는 **ChatGPT·Claude로 열기에만** 적용된다. "복사하기"와 ".md 다운로드" 결과에는 어떤 프롬프트 텍스트도 포함되지 않는다.

### 직접 입력 프롬프트는 1회용
"직접 입력" 텍스트 인풋의 내용과 프롬프트 영역의 선택 상태(프리셋 또는 직접 입력)는 페이지 새로고침·탭 닫기 시 사라진다. localStorage·서버에 저장하지 않는다.

### 단일 선택
프롬프트 영역에서는 프리셋 3종과 "직접 입력" 중 한 항목만 선택할 수 있다. 같은 항목을 다시 누르거나 별도 해제 동작으로 "선택 안 함" 상태로 되돌릴 수 있어야 한다.

### 성능
정상적으로 도달 가능한 일반 웹 페이지(예: 블로그 글 한 편)는 화살표 버튼을 누른 시점부터 결과가 화면에 보이기까지 5초 이내에 완료되어야 한다. 그 이상 걸리면 "변환 실패"로 처리한다.

### 반응형
모바일·데스크톱 너비 모두에서 모든 시나리오가 동일하게 동작해야 한다. 입력 필드, 결과 영역, 내보내기 버튼, 프롬프트 영역, 다크모드 토글이 화면을 벗어나거나 가려지지 않는다.

## 의존성

- 본문 추출은 **defuddle** 라이브러리를 사용한다 (요구사항에서 지정됨, https://defuddle.md/docs)
- ChatGPT·Claude의 공식 웹 인터페이스가 새 탭으로 도달 가능해야 한다 (URL 자체는 plan.md에서 확정)
- 클립보드 복사를 위해 사용자 브라우저의 Clipboard API 권한 정책이 허용되어야 한다 (사용자 클릭 직후의 write는 일반적으로 권한 없이 동작)
